## 一、项目概述

**无敌 (Invencia)** 是一款叙事驱动的修仙文字 RPG。AI"天道"智能体全程驱动叙事、NPC 对话、战斗判定与世界推进。玩家通过自由文本输入做出抉择，每段叙事以开放式钩子结尾。摒弃数值面板，用文字描摹强弱。出身由 AI 随机生成，每个玩家的开局都是"盲盒"。永久死亡机制，死亡角色进入排行榜成为世界记忆。

## 二、关键决策记录

| # | 决策 | 详情 |
|---|---|---|
| 1 | **项目名** | 无敌，英文名 Invencia |
| 2 | **交互方式** | 自由文本输入，玩家输入决策性内容，AI 生成叙事，结尾返回开放式钩子 |
| 3 | **天道职责** | NPC 对话、战斗判定、世界推进、命运/随机事件/渡劫 |
| 4 | **玩法重心** | 重叙事（互动小说），轻数值。战力改用文字描述，预设强弱层级但无数字 |
| 5 | **境界体系** | 传统修仙：炼气→筑基→金丹→元婴→化神→渡劫→大乘→飞升（硬设定） |
| 6 | **文气强弱** | 仅用于同境界内微妙区分（初窥门径/渐入佳境/炉火纯青/登峰造极），纯叙事表达 |
| 7 | **出身系统** | 玩家仅选择性别，其余全部由 AI 随机生成（身世、地点、执念、机缘） |
| 8 | **差异维度** | 出身随机 × 性格演绎 × 玩家选择 × 随机事件 × 因果记忆 |
| 9 | **持久性** | 长期游玩，跨 session 保存进度 |
| 10 | **死亡机制** | 永久死亡，不可复活。死亡后生成墓志铭，角色进入排行榜 |
| 11 | **社交** | 纯单机，仅排行榜（飞升碑/陨落录/因果榜/传奇榜）提供轻度社交 |
| 12 | **产品形态** | 先做纯网页，效果显著再做客户端 |
| 13 | **后端** | Python FastAPI |
| 14 | **前端** | 纯 HTML + CSS + Vanilla JS，零构建步骤，UI 追求上线级品质 |
| 15 | **数据库** | SQLite（单文件零配置） |
| 16 | **AI 模型** | DeepSeek v4 Pro/Flash、通义千问系列（国产优先） |
| 17 | **用户系统** | 需要注册/登录/存档，尽可能简单 |
| 18 | **世界设定** | 固定修仙世界"九域"，有既定地图、宗门、历史。玩家从不同时间线/地点切入 |
| 19 | **infomation 存储** | 全叙事描述，不存枚举值。每字段都是一段画面级叙事文本，AI 可直接读入上下文 |
| 20 | **infomation 维护** | 由 AI 在每次对话后自主决定增量更新，随对话持续迭代 |

## 三、数据模型（已确定）

### t_user（用户表）

| 字段 | 类型 | 说明 |
|---|---|---|
| id | BIGINT | 主键，自增 |
| username | VARCHAR(255) | 用户名 |
| nickname | VARCHAR(255) | 用户昵称 |
| password_hash | VARCHAR(255) | 密码（哈希） |
| role | VARCHAR(255) | 用户身份 |
| created_at | TIMESTAMP | 创建时间 |
| updated_at | TIMESTAMP | 更新时间 |
| del_flag | SMALLINT | 删除标识 |

### t_character（角色表）

| 字段 | 类型 | 说明 |
|---|---|---|
| id | BIGINT | 主键，自增 |
| user_id | BIGINT | 用户 ID |
| name | VARCHAR(255) | 角色昵称 |
| gender | SMALLINT | 性别（玩家自选） |
| infomation | JSON | 角色信息，全叙事描述，AI 持续迭代维护 |
| created_at | TIMESTAMP | 创建时间 |
| updated_at | TIMESTAMP | 更新时间 |
| del_flag | SMALLINT | 删除标识 |

infomation 字段（10 个子字段，均为叙事描述）：ealm, power, origin, obsession, location, dao_heart, causality, elationships, inventory, summary

### t_message（消息表）

| 字段 | 类型 | 说明 |
|---|---|---|
| id | INT | 主键，自增 |
| conversation_id | BIGINT | 会话（角色）ID |
| type | VARCHAR(255) | assistant / user / system |
| content | TEXT(65535) | 消息内容 |
| metadadta | JSON | 元数据（事件标记等） |
| created_at | TIMESTAMP | 创建时间 |
| updated_at | TIMESTAMP | 更新时间 |
| del_flag | SMALLINT | 删除标识 |

> 一张表双重职责：会话记忆 + 事件记录

## 五、MVP 范围

**包含**：
- 注册/登录
- 创建角色（选性别，其余天道随机生成）
- 全境界自由游玩（不限境界范围）
- 天道 AI 驱动叙事，infomation 自动维护
- 对话历史持久化（关闭浏览器可继续）
- 上线级 UI（黑底古风，玉简/卷轴风格角色面板）

**不包含**：
- 排行榜
- 永久死亡判定（MVP 渡劫仅叙事）
- 多角色并行

## 九、架构思路

`
浏览器（HTML/CSS/JS）
    ↕ HTTP
FastAPI 后端
    ├── auth 模块（注册/登录/JWT）
    ├── game 模块（角色创建、叙事生成、infomation 维护）
    └── leaderboard 模块（排行榜，MVP 后实现）
    ↕
SQLite（t_user / t_character / t_message）
    ↕
AI API（DeepSeek v4 / 通义千问）
`

核心循环：用户输入 → 取最近 N 条消息 + infomation → 构建 System Prompt → 调用 AI → 解析叙事 + infomation 增量 → 写入 t_message + 更新 t_character.infomation → 返回叙事 + 钩子
